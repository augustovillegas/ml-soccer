# Research docs

Esta carpeta guarda investigacion tecnica, analisis de librerias, comparativas y reportes de referencia que sirven para decidir como implementar la extraccion, transformacion o modelado dentro del proyecto.

Uso esperado:

- guardar reportes de investigacion profunda sobre librerias o fuentes de datos
- registrar hallazgos tecnicos previos a una implementacion
- centralizar documentacion de soporte que no corresponde a notebooks ni a codigo ejecutable

Convencion:

- usar nombres descriptivos en Markdown
- mantener estos documentos como referencia de arquitectura y de decisiones tecnicas
